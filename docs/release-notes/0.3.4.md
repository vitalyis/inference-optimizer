# Release Notes: v0.3.4

Patch release for a real OpenClaw install regression: `/optimize` and the related skill commands could stop working after a bot update because workspace wiring remained correct while the required script paths were no longer pre-approved in `exec-approvals.json`.

## What changed

- `scripts/setup.sh --apply` now manages the required path-specific approval entries for both `main` and `whatsapp` agents.
- `scripts/verify.sh` now checks that those approval entries exist, in addition to checking workspace command wiring.
- Reapplying the skill now restores the complete runtime surface for:
  - `/audit`
  - `/optimize`
  - `/preflight`
  - `purge sessions`

## Why this release matters

The prior package could look correctly installed while still failing at runtime with approval prompts for the main advertised commands. This patch closes that gap by making approval state part of the managed installation and part of verification.
